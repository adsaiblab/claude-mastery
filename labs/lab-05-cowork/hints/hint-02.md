# Hint 02 — Lab 05

Structure de `plan.md` (UltraPlan) :

```markdown
# Plan — rate-limit

## Objectif
[1 phrase]

## Hypothèses
- [liste]

## Étapes
1. ...
2. ...

## Risques
- [risque → mitigation]

## Critères de succès
- Tests passent.
- Complexité O(1) amortie par requête.
- Pas de fuite mémoire au-delà de 10k IPs.
```

L'UltraReview doit **challenger** : hypothèses fausses, étapes manquantes, critères laxistes. Il produit `plan.v2.md` avec les corrections.
