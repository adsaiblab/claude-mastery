<Situation>
  Une feature arrive en fin de sprint. Tu veux qu'elle passe par un vrai
  pipeline qualité : un planner qui cadre, un implementer qui applique, un judge
  qui relit avant commit — et un **hook Stop qui refuse la fin de tour si la
  qualité n'est pas atteinte**. Dans ce lab, tu cables cette pipeline de bout en
  bout.
</Situation>

## Objectif

Livrer un setup où un prompt utilisateur enclenche automatiquement :

1. **Planner** (subagent, opus) → produit un `plan.md` structuré.
2. **Implementer** (subagent, sonnet, isolation worktree) → applique le plan.
3. **Judge** (subagent, opus, read-only) → score 0-10 avec verdict.
4. **Hook Stop** → si score < 7, renvoie stderr à l'agent avec les concerns, exit 2.
5. **Slash-command `/pipeline`** → encapsule le tout en une commande unique.

## Démarrer

```bash
cd labs/lab-03-pipeline
./setup.sh              # crée work/ avec repo exemple + starter
cd work/
```

Tu dois créer :

- `.claude/agents/planner.md`, `implementer.md`, `judge.md`
- `.claude/commands/pipeline.md` (slash-command custom)
- `.claude/hooks/quality-gate.sh` (hook Stop)
- `.claude/settings.json` (déclaration hook + permissions)

Le script exécute le scénario complet :

1. Lance `/pipeline "ajoute une fonction `divide(a, b)` avec gestion /0"`.
2. Vérifie que `plan.md` a été créé et contient les sections attendues.
3. Vérifie que l'implementer a produit du code dans un worktree isolé.
4. Force un cas **avec bug volontaire** → le judge doit scorer < 7 → le hook Stop doit renvoyer exit 2.
5. Re-lance avec fix → score ≥ 7 → exit 0.

## Ce qu'on vérifie vraiment

1. Les 3 fichiers `.claude/agents/*.md` existent avec frontmatter `name/description/tools/model` valides.
2. `planner.md` a `tools: Read, Grep, Glob` (pas d'écriture).
3. `judge.md` a `tools: Read, Grep` (strictement read-only) + `model: opus`.
4. `implementer.md` a `isolation: worktree`.
5. `.claude/commands/pipeline.md` existe et enchaîne les 3 subagents dans l'ordre.
6. `.claude/hooks/quality-gate.sh` est **exécutable** et déclaré sur l'événement `Stop`.
7. Quand le judge retourne un score < 7, le hook **exit 2** avec stderr listant les concerns.
8. Quand le score ≥ 7, le hook **exit 0** sans interrompre.
9. Pas de secret dans `.claude/settings.json`.
