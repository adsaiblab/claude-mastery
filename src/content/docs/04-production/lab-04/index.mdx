<Situation>
  Tu vas livrer un workflow GitHub Actions qui review chaque PR du repo, poste
  un commentaire structuré (score 0-10 + concerns ligne par ligne), et **bloque
  le merge** via un required check si le score est insuffisant. Une review
  auto-déclenchée, déterministe, pour soulager les reviewers humains sur les
  basiques.
</Situation>

## Objectif

Livrer un setup où chaque PR ouverte ou mise à jour déclenche :

1. Un job GitHub Action qui checkout la PR.
2. Un run `anthropics/claude-code-action` avec un prompt structuré + tools restreints (lecture + `gh pr comment`).
3. Un commentaire unique (mis à jour en place si la PR est push à nouveau).
4. Un exit code non-zéro si score < 7, qui fait **échouer** un check required côté branch protection.

## Démarrer

```bash
cd labs/lab-04-ci
./setup.sh              # crée work/ avec repo démo + workflow starter
cd work/
```

Secrets à configurer dans le repo GitHub (Settings → Secrets and variables → Actions) :

- `ANTHROPIC_API_KEY`

Tu dois créer :

- `.github/workflows/claude-review.yml` (workflow complet).
- Prompt de review avec scoring structuré (extractible par regex).
- Concurrence + skip-CI marker pour éviter les boucles.

## Ce qu'on vérifie vraiment

1. Le workflow `.github/workflows/claude-review.yml` se déclenche sur `pull_request: [opened, synchronize]`.
2. Permissions **minimales** : `contents: read` + `pull-requests: write`. Pas de `write` sur `contents`.
3. `allowed_tools` est restreint à la lecture + `gh pr comment`/`gh pr review` — pas de wildcard.
4. `timeout_minutes` ≤ 10.
5. Le prompt contient un format **extractible** (ex: `Score: N/10`) que le step suivant parse pour décider exit 0/1.
6. Le commentaire posté est **unique** et mis à jour en place (pas un nouveau commentaire par push).
7. Un marqueur `[skip ci]` est inclus dans les commits éventuels de Claude pour éviter une boucle.
8. Le secret `ANTHROPIC_API_KEY` n'apparaît jamais dans les logs (pas de `echo $ANTHROPIC_API_KEY`).
9. `concurrency.group` est défini pour qu'un nouveau push à la PR annule le run précédent.
