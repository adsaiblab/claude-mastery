## Objectif

Livrer un module Python qui combine :

1. **Cron quotidien** (`daily_digest.py`) — produit un digest des PRs du jour, idempotent.
2. **Endpoint webhook** (`webhook.py`, FastAPI) — trigger manuel avec HMAC + idempotence partagée.
3. **`lib/idempotence.py`** — clé déterministe + store SQLite.
4. **`lib/retry.py`** — décorateur `@retry(max=3)` avec backoff exponentiel + jitter.
5. **`lib/dlq.py`** — push vers `dlq.jsonl` après N échecs.
6. **Tests pytest** qui vérifient idempotence, retry, DLQ, HMAC.

Le script :

## Ce qu'on vérifie vraiment

1. La clé d'idempotence est **déterministe** (même input → même clé). Pas de timestamp dans la clé.
2. Le store idempotence est durable (SQLite) — pas un dict en mémoire.
3. Le retry a **du jitter** (sinon les retries se synchronisent en cas de panne globale).
4. La DLQ est en JSONL append-only — chaque ligne est un événement reproductible.
5. Le webhook utilise `hmac.compare_digest` (pas `==`).
6. Le webhook **rejette** sans révéler la cause exacte (juste 401 + message générique).
7. La crontab est une **string-template** : pas de chemin codé en dur incompatible.
8. Les tests **isolent** le store (pas de fuite entre tests via fixture `tmp_path`).
