<Situation>
  Lab final : tu vas livrer **ton propre système** d'agents sur un module
  existant. Subagents typés (planner, implementer, judge), hook Stop pour
  garde-fou, mémoire typée pour la continuité, cycle UltraPlan/UltraReview
  comparé à un baseline one-shot. Tu finis le parcours avec un setup
  reproductible que tu peux poser sur ton vrai projet demain matin.
</Situation>

## Objectif

1. Choisir un module non trivial (50-200 lignes) à modifier ou refactorer.
2. Produire une **baseline one-shot** : prompt unique, sans cycle, mesurer.
3. Exécuter le **cycle complet** UltraPlan → UltraReview → Implement → Judge avec subagents dédiés.
4. Comparer les deux livrables sur grille fixe et conclure.

## Démarrer

```bash
cd labs/lab-05-cowork
./setup.sh              # crée work/ avec le module à modifier + grille de jugement
cd work/
```

Tu dois créer dans `.claude/agents/` :

- `ultra-planner.md` (Opus, tools lecture seule).
- `ultra-reviewer.md` (Opus, tools lecture seule).
- `ultra-implementer.md` (Sonnet, tools full + `isolation: worktree`).
- `ultra-judge.md` (Opus, tools lecture seule, **prompt explicite** : pas de hérissage de la conv).

Et dans `.claude/`:

- `hooks/check-score.sh` qui parse `verdict.md` et exit 2 si score < 7.
- `settings.json` avec hook Stop branché sur `ultra-judge` + `check-score.sh`.
- `memory/MEMORY.md` + au moins une mémoire `project_<sujet>.md` de continuité.

Le script vérifie :

1. Les 4 subagents existent dans `.claude/agents/` avec frontmatter conforme (name, description, tools, model).
2. `settings.json` branche le hook Stop sur `ultra-judge` + `check-score.sh`.
3. Les 4 artefacts sont produits : `plan.md`, `revised-plan.md`, `verdict-baseline.md`, `verdict-cycle.md`.
4. `verdict-cycle.md` score > `verdict-baseline.md` score (sinon le cycle n'apporte rien — refais).
5. `MEMORY.md` indexe au moins un fichier mémoire typé.
6. `diff.md` contient un tableau comparatif quantifié (4 colonnes minimum).

## Ce qu'on vérifie vraiment

1. Les 4 rôles sont **distincts** (pas de planner qui code, pas de judge qui voit la conv).
2. Le hook Stop est **réversible** (on peut le désactiver sans casser le projet).
3. Le score cycle bat le score baseline d'au moins **2 points** sur 10.
4. La mémoire typée indexe correctement (frontmatter, pointeur dans MEMORY.md).
5. Le `diff.md` montre que le **temps humain** post-livraison est inférieur côté cycle (le critère qui justifie le coût).
6. Les artefacts sont **suffisants** pour qu'un autre dev reprenne le travail froid.
