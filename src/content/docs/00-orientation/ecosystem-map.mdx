## Concept — Les 5 couches

L'écosystème Claude s'empile en **5 couches**, de la plus fondamentale à la plus orchestrée :

<div style="overflow-x: auto;">

| # | Couche | Réponse à la question | Exemples concrets |
|---|--------|-----------------------|-------------------|
| **1** | Modèles | *Quel cerveau ?* | Opus 4.7, Sonnet 4.6, Haiku 4.5 |
| **2** | Capacités | *Quoi faire nativement ?* | Tool use, vision, extended thinking, caching |
| **3** | Interfaces | *D'où je lui parle ?* | API, Claude Code (CLI + IDE), claude.ai, apps |
| **4** | Orchestration | *Comment l'étendre ?* | Subagents, Skills, MCP, Hooks, Plugins |
| **5** | Déploiement | *Comment l'industrialiser ?* | GitHub Actions, Agent SDK, sandboxing, Managed Agents |

</div>

Chaque niveau du parcours **monte d'une couche** :

- **Niveau 1** — Fondations = couches 1 + 2 (comprendre ce qu'est un modèle et une boucle agentique).
- **Niveau 2** — CLI Mastery = couche 3 (tout ce qui se passe dans le terminal).
- **Niveau 3** — Multi-agents = couche 4 (subagents, MCP, plugins).
- **Niveau 4** — Production = couche 5 (CI/CD, SDK, sandboxing).
- **Niveau 5** — Expert = recompose les 5 couches en patterns avancés.

## Détail — Couche 4 : orchestration

C'est **ici que la complexité réelle commence**. Cinq mécanismes distincts, souvent confondus :

  />
  <FlipCard
    client:visible
    frontLabel="Orchestration"
    backLabel="Définition"
    front="Hook"
    back="Shell command qui s'exécute en réponse à un événement (PreToolUse, Stop, UserPromptSubmit). Garde-fous et automations."
  />
  <FlipCard
    client:visible
    frontLabel="Orchestration"
    backLabel="Définition"
    front="Plugin"
    back="Bundle distribuable regroupant skills + hooks + slash commands + MCP servers. Partage d'équipe."
  />
</div>

**Règle de décision :**

- Tâche ponctuelle isolée → **Subagent**.
- Procédure réutilisable documentée → **Skill**.
- Accès à un outil externe → **MCP**.
- Policy à appliquer automatiquement → **Hook**.
- Ensemble cohérent à partager → **Plugin**.

## Détail — Couche 5 : déploiement

| Composant | Rôle |
|-----------|------|
| **GitHub Actions** | Déclenche Claude sur PR / issue / schedule |
| **Agent SDK (Python / TypeScript)** | Construction d'agents custom hors CLI |
| **Sandboxing** | Exécution isolée (container, VM) pour actions risquées |
| **Routines** | Agents long-running pilotés par un orchestrateur |
| **Managed Agents** | Déploiement d'agents hébergés par Anthropic |
