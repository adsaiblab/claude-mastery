## Concept — le context comme budget

Le context window, c'est l'**espace de travail** du modèle. À chaque tour, il contient :

- Le **system prompt** (fourni par Claude Code).
- Ton **CLAUDE.md** et les mémoires chargées.
- L'**historique de conversation** (chaque tour précédent).
- Les **outputs d'outils** (Read, Bash, Grep…).
- Ton **prompt courant**.

Tout ça est compté en **tokens**. Un modèle a une limite — aujourd'hui ~200k tokens pour Opus/Sonnet (~500k en mode extended).

**Règle mentale :** c'est comme la RAM. Quand c'est plein, ça swap (troncature silencieuse) et ça ralentit.

## La règle des 60%

Au-delà de **~60% de remplissage**, la qualité du modèle commence à se dégrader :

- Il "oublie" des éléments des premiers tours.
- Il réinvente des détails qu'on lui avait donnés.
- Il mélange des fichiers différents.

**Cible pratique :** reste en dessous de 60%. Au-dessus de 80%, ton prochain prompt est une perte de temps — compacte ou clear.

Claude Code t'affiche l'usage dans l'UI (barre de remplissage en bas de l'écran, ou via `/status`).

## Les 4 leviers — quand utiliser quoi

| Commande | Effet | Quand l'utiliser |
|----------|-------|------------------|
| `/compact` | Résume l'historique, garde l'essentiel. | ~70% rempli, tu veux continuer la même tâche. |
| `/clear` | Vide la conversation, garde CLAUDE.md. | Tu passes à un sujet **sans rapport**. |
| `/btw` | Injecte un message système pour recentrer. | L'agent a dérivé, tu veux le recadrer sans perdre le contexte. |
| Nouvelle session | Repart de zéro (nouveau jsonl). | Tu changes de projet, nouveau reset complet. |

### `/compact` — la compaction

Claude résume l'historique de la session en gardant : objectif courant, décisions prises, fichiers touchés, erreurs rencontrées. Coûte quelques milliers de tokens pour en libérer des dizaines de milliers.

**Astuce** : tu peux personnaliser la compaction en ajoutant une directive dans ton `CLAUDE.md` :

```markdown
## Compaction
Préserver toujours : objectif courant, liste des fichiers touchés, commandes
de test utilisées. Compresser : explications détaillées, lectures exploratoires.
```

### `/clear` — le reset complet

Vide la conversation. `CLAUDE.md` et la mémoire **restent** chargés. Utile quand tu changes de sujet et que l'historique précédent n'aide plus — au contraire, il pollue.

### `/btw` — l'injection

Message système ponctuel. Pratique pour :

- "BTW, toutes les commandes git doivent passer par la CI, ne commit pas localement."
- "BTW, ignore les fichiers `.generated.ts`."
- "BTW, passe en mode plan avant d'éditer quoi que ce soit."

C'est une télécommande pour recentrer l'agent sans redémarrer.

## Fork / nouvelle session — quand reset

- `claude --fork-session` — crée une copie de la session courante à un état donné. Utile pour **explorer deux pistes** en parallèle sans les mélanger.
- Nouvelle session (`claude` nu) — repart à zéro. Préférable quand : tu changes de projet, tu as fini une grosse tâche, ou tu sens que la session est "polluée".

## Interaction avec les env vars

- `CLAUDE_CODE_MAX_CONTEXT_TOKENS` — forcer une taille max plus petite (utile pour tester que ton code marche sous contrainte).
- `CLAUDE_CODE_AUTO_COMPACT=false` — désactiver la compaction automatique (déconseillé en pratique).

## Démo — un context qui déborde

```
[~15% rempli] ✓ session fraîche, réponses rapides et précises
[~45% rempli] ✓ toujours nickel
[~65% rempli] ⚠️ commence à oublier des détails anciens → /compact
[~85% rempli] ✗ hallucinations, réponses vagues → trop tard, /clear ou nouvelle session
```

## Context engineering — le réflexe de pro

Ne considère pas le contexte comme quelque chose qui **se remplit passivement** — considère-le comme quelque chose que tu **composes** :

1. **Minimiser** ce que tu charges (CLAUDE.md court, mémoire scopée, pas de gros dumps).
2. **Injecter** précisément ce qui est utile au moment où c'est utile (`/btw`, fichiers pointés explicitement).
3. **Compacter** agressivement à 60-70%.
4. **Forker** quand tu veux explorer sans polluer.
5. **Archiver dans MEMORY.md** ce qui doit survivre entre sessions.

Le Niveau 5 a un module dédié ([Context Engineering](/05-expert/context-engineering/)) qui pousse ces patterns plus loin.
