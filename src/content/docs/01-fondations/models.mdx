## Concept — la famille Claude 4.X

Trois modèles, un seul principe : **plus on paie, plus on raisonne**. Il faut choisir en fonction de ce qu'on demande.

## Règle de décision

```
Tâche → quel modèle ?
│
├─ Raisonnement long / jugement / architecture
│    → Opus
│
├─ Implémenter un plan / refacto / tests
│    → Sonnet
│
└─ Classer / trier / extraire / paralléliser
     → Haiku
```

**Pattern canonique multi-modèles :** Haiku classe 20 documents en parallèle → Sonnet en extrait les enjeux → Opus juge et décide. Coût total : une fraction d'un seul appel Opus sur l'ensemble.

## Comment choisir depuis le CLI

```bash
# À l'ouverture
claude --model opus              # alias
claude --model sonnet            # alias
claude --model haiku             # alias
claude --model claude-opus-4-7   # ID complet

# Pendant une session
/model opus
/model sonnet
```

Tu peux aussi fixer le modèle **par subagent** dans son frontmatter (cf. Niveau 3).

## Démo — comparer sur une vraie tâche

Même question, trois modèles :

```
> Extrais les dépendances listées dans ce package.json.

[Haiku]    1.8s  — liste correcte, pas de commentaire.
[Sonnet]   3.1s  — liste correcte + détecte un semver exotique sur `react`.
[Opus]     4.5s  — liste + détection + recommandation de mettre à jour 2 deps.
```

Pour une extraction simple, **Haiku suffit**. Pour une review de PR, tu veux Opus (ou au moins Sonnet avec plus de contexte).

## Adaptive thinking

Depuis Claude 4.5, le modèle ajuste automatiquement la profondeur de raisonnement selon la complexité perçue. Tu n'as plus à activer `extended_thinking` manuellement dans la plupart des cas interactifs — le CLI s'en charge. **Mais tu peux toujours le forcer** pour des problèmes durs via les paramètres API ou le flag `--thinking-budget` côté SDK.

## Fallback automatique

Si le modèle demandé est indisponible (rate-limit, incident), Claude Code peut basculer sur un modèle proche. Voir `CLAUDE_CODE_ENABLE_FALLBACK` dans la [référence env vars](/reference/env-vars/). **En prod, c'est un piège** si tu attendais une qualité spécifique — mieux vaut échouer explicitement.
