<Situation>
  Ton équipe a deux règles non négociables : (1) aucun `git push` direct sur `main`,
  (2) tout fichier édité doit être auto-formaté (prettier JS/TS/MD, black Python).
  Jusqu'ici, tu espérais que chaque dev s'en souvienne. Dans ce lab, tu les
  **gravées dans le marbre** sous forme de hooks qui marchent même quand Claude
  n'a pas lu ton CLAUDE.md.
</Situation>

## Objectif

Écrire deux hooks qui passent les trois scénarios suivants :

1. **`git push origin main`** → bloqué avec un message d'erreur clair sur stderr (exit 2).
2. **`git push origin feature/lab-02`** → autorisé (exit 0).
3. **N'importe quelle autre commande** (`ls`, `npm test`, …) → pas d'intervention (exit 0).

Plus un hook PostToolUse qui formatte automatiquement les fichiers `.js/.ts/.json/.md/.py` après chaque Edit ou Write.

## Démarrer

```bash
cd labs/lab-02-hooks
./setup.sh              # crée work/ avec package.json starter
cd work/
```

Tu dois créer :

- `.claude/settings.json` avec les déclarations des hooks.
- `.claude/hooks/validate-push.sh` (exécutable).
- `.claude/hooks/format-file.sh` (exécutable).

## Ce qu'on vérifie vraiment

1. `.claude/settings.json` déclare un `PreToolUse` (matcher `Bash`) et un `PostToolUse` (matcher `Edit|Write`).
2. Les scripts `.claude/hooks/*.sh` existent et sont **exécutables** (`chmod +x`).
3. `validate-push.sh` **exit 2** sur `git push origin main` et écrit un message sur **stderr**.
4. `validate-push.sh` **exit 0** sur `git push origin feature/lab-02`.
5. `validate-push.sh` **exit 0** sur toute commande qui n'est pas un `git push`.
6. `format-file.sh` appelle `prettier --write` pour `.js/.ts/.json/.md`, `black` pour `.py`.
7. Un formatter manquant ne **bloque pas** l'édition (exit 0 même si prettier absent).
8. Le JSON de `settings.json` est valide (`jq . .claude/settings.json` passe).
