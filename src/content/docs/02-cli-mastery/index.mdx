## Pourquoi ce niveau

Au Niveau 1, tu as le **modèle mental**. Maintenant, tu acquiers les **réflexes moteurs** qui font la différence entre "utilisateur occasionnel" et "power user" :

- Les bons **flags** au lancement (et savoir lesquels existent).
- Les bonnes **slash commands** en session (sans chercher dans la doc).
- **Checkpointing** pour oser les refactors risqués (rewind en un raccourci).
- **Hooks** pour automatiser ce qui doit être automatique.
- **Permission modes** pour doser vitesse vs sécurité selon la tâche.
- **Sessions nommées, forkées, taggées** — ton historique devient navigable.

## Lab 02 — Hooks pipeline

[Lab 02](/02-cli-mastery/lab-02/) — créer un hook `PreToolUse` qui bloque les `git push` directs sur `main` ET un hook `PostToolUse` qui auto-formate les fichiers édités (prettier/black selon l'extension). Validation scénarisée qui injecte de vrais `tool_input` pour tester les deux branches.

## Après ce niveau, tu sauras

- Ouvrir Claude avec les bons flags pour le contexte exact (model, permissions, tools restreints).
- Enchaîner `Shift+Tab` entre permission modes sans hésiter.
- Récupérer d'une dérive de Claude en 2 secondes via `Esc+Esc` ou `/rewind`.
- Écrire un hook bash qui bloque ou autorise une action selon des critères précis.
- Retrouver une session d'il y a 3 semaines via nom ou tag.
